# API Documentation

Base URL: `http://localhost:8765`

All authenticated endpoints require `Authorization: Bearer <access_token>` header.

---

## Auth

| Method | Path | Body | Response |
|--------|------|------|----------|
| POST | `/api/auth/register` | `{username, email, password}` | `{id, username, email, is_active, created_at}` |
| POST | `/api/auth/login` | `{username, password}` | `{access_token, refresh_token, token_type}` |
| POST | `/api/auth/refresh` | `{refresh_token}` | `{access_token, token_type}` |
| GET | `/api/auth/me` | - | `{id, username, email, is_active, created_at}` |

## Chat

LLM-driven tool calling. The model decides whether to invoke `search_knowledge` (RAG) or `search_memory` (conversation history).

| Method | Path | Body | Response |
|--------|------|------|----------|
| POST | `/api/chat` | `{message, conversation_id?, system_context?}` | `{response, conversation_id, sources}` |
| POST | `/api/chat/stream` | `{message, conversation_id?}` | SSE stream (`data: ...\n\n`) |

Example:
```json
// POST /api/chat
{"message": "OFDM循环前缀的作用是什么？", "conversation_id": 1}

// Response
{
  "response": "循环前缀（CP）用于消除OFDM中的符号间干扰...",
  "conversation_id": 1,
  "sources": [{"content": "...", "score": 0.85, "source": "4_无线信道的信道容量.md"}]
}
```

## Conversations

| Method | Path | Params | Response |
|--------|------|--------|----------|
| POST | `/api/conversations` | - | `{id, title, created_at}` |
| GET | `/api/conversations` | - | `[{id, title, message_count, ...}]` |
| GET | `/api/conversations/{id}` | `?limit=50&before={msg_id}` | `{id, title, messages[], has_more, total}` |
| DELETE | `/api/conversations/{id}` | - | `{status: "deleted"}` |

Messages are cached in Redis (30min TTL, cache-on-write). Pagination uses `before={msg_id}`.

## Code Execution

| Method | Path | Body | Response |
|--------|------|------|----------|
| POST | `/api/code/execute` | `{code, language?}` | `{success, stdout, stderr, exit_code, images[]}` |
| POST | `/api/code/generate` | `{description, language?}` | `{code, result, original_code?, fix_attempts?}` |

- `images[]` contains base64-encoded PNG images generated by matplotlib
- `/api/code/generate` auto-retries with LLM debugging up to 3 times
- `SANDBOX_MODE=docker` for production isolation

## Knowledge Base

| Method | Path | Params/Body | Response |
|--------|------|-------------|----------|
| GET | `/api/knowledge/search` | `?query=...&top_k=5` | `{results: [{content, score, source}]}` |
| POST | `/api/knowledge/upload` | FormData `file` (.pdf/.md/.txt/.py) | `{filename, chunks, status}` |
| GET | `/api/knowledge/papers` | - | `{papers: [{source, chunks}], count}` |
| DELETE | `/api/knowledge/papers/{id}` | - | `{status, paper_id, chunks_removed}` |

Uses hybrid search (BM25 + dense vector) with optional LLM reranking via ZhipuAI.

## Papers (PDF Reading)

| Method | Path | Body | Response |
|--------|------|------|----------|
| POST | `/api/papers/upload` | FormData `file` (.pdf) | `{paper_id, filename, summary, page_count}` |
| GET | `/api/papers` | - | `{papers: [{id, filename, summary, ...}]}` |
| GET | `/api/papers/{id}` | - | `{id, filename, full_text, summary, messages[]}` |
| POST | `/api/papers/{id}/chat` | `{message}` | `{response, paper_id}` |
| DELETE | `/api/papers/{id}` | - | `{status, paper_id}` |

Paper content is extracted via pdfplumber. The system prompt includes the paper's full text (truncated to 30K chars head+tail for long papers).
