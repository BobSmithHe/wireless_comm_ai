# 第8章 无线信道中的编码

## 8.1 引言：编码如何对抗无线信道的挑战？

在AWGN信道中，信道编码的主要作用是**纠正随机错误**。然而，无线信道引入了新的、更严峻的挑战：**衰落**导致的**突发错误**和**低信噪比**。本章探讨专门为应对这些挑战而设计的信道编码技术。

无线信道编码的核心目标是在不增加带宽或功率的前提下，通过引入**冗余**来提高通信的**可靠性**。与第七章讨论的分集（一种信号处理技术）不同，信道编码是一种**信息处理技术**，两者结合可产生巨大协同增益。

**关键挑战与编码对策**：
1.  **深衰落导致长突发错误** → **交织**：将错误打散，使信道近似为随机错误信道。
2.  **功率和带宽受限** → **功率高效和频谱高效的编码**：如编码调制、低密度校验码。
3.  **信道时变** → **自适应编码调制**：根据信道条件动态调整编码和调制方案。
4.  **高误码率要求** → **强大纠错码**：如Turbo码、LDPC码，以接近香农极限。

## 8.2 线性分组码基础

### 8.2.1 核心概念
线性分组码将 \(k\) 个信息比特映射为 \(n\) 个编码比特，记为 \((n, k)\) 码。其核心是**线性结构**：任意两个码字的线性组合仍是码字。

- **生成矩阵** \(G\) （\(k \times n\)）：信息矢量 \(\mathbf{u}\) 编码为码字 \(\mathbf{c} = \mathbf{u}G\)。
- **校验矩阵** \(H\) （\((n-k) \times n\)）：满足 \(GH^T = 0\)，用于检测错误，\(H\mathbf{c}^T = 0\) 当且仅当 \(\mathbf{c}\) 是有效码字。
- **最小汉明距离** \(d_{\min}\)：任意两个不同码字之间不同的最小比特数。它决定了码的纠错能力：可纠正最多 \(t = \lfloor (d_{\min}-1)/2 \rfloor\) 个随机错误。

### 8.2.2 汉明码
- 最简单的纠单错完备码，参数为 \((2^m-1, 2^m-1-m)\)，\(d_{\min}=3\)。
- 是许多更复杂编码的基础构件，但在无线信道中单独使用能力不足，因其无法应对突发错误。

### 8.2.3 循环码与BCH码
- **循环码**：码字的任何循环移位仍是码字，可用多项式表示和编码，便于硬件实现（用移位寄存器）。
- **BCH码**：一类强大的循环码，可纠正多个随机错误。其参数灵活，是**里德-所罗门码**（非二进制BCH码）的基础，后者是纠正突发错误的利器，常用于级联码的外码。

## 8.3 卷积码

与分组码不同，卷积码具有**记忆性**，其输出不仅取决于当前输入，还取决于之前的输入。这使其在低复杂度下能获得良好的性能。

### 8.3.1 编码器与状态
卷积码由 \(k\) 个输入、\(n\) 个输出、约束长度 \(K\) 的移位寄存器定义，码率 \(R = k/n\)。
- 编码器状态数：\(2^{K-1}\)。
- 常用**网格图**表示所有可能的状态转移路径，每个分支对应一个输出符号。

### 8.3.2 维特比译码
最大似然序列检测算法，在网格图上寻找与接收序列**欧氏距离**（软判决）或**汉明距离**（硬判决）最小的路径。

- **核心操作**：递归地计算和比较到达每个状态的**路径度量**，保留最优的**幸存路径**。
- **软判决增益**：利用解调器输出的多比特置信度信息，相比硬判决可带来约2-3 dB的性能增益，对无线系统至关重要。
- **译码深度**：通常为5-10倍约束长度，以保证幸存路径以高概率汇聚。

卷积码因其良好的性能和适中的复杂度，在2G（GSM）、3G（CDMA2000）等系统中被广泛用作核心编码。

## 8.4 级联码

为应对无线信道中混合的随机和突发错误，**级联码**应运而生。它由内外两级编码串联构成，中间加一个**交织器**。

- **经典结构**：外码采用**里德-所罗门码**（擅长纠突发错误），内码采用**卷积码**（擅长纠随机错误）。
- **交织器**：将内译码器输出的突发错误打散，变成外码可处理的随机错误。
- **性能**：通过级联，可以用两个中等复杂度的码实现接近长分组码的性能。此结构是**Turbo码**思想的前身。

## 8.5 Turbo码：逼近香农极限的革命

Turbo码是通信史上的里程碑，首次以可接受的复杂度实现了接近香农极限的性能。

### 8.5.1 编码结构
- 两个（或多个）**系统递归卷积码**编码器并行或串行排列。
- 关键：在编码器之间插入一个**随机交织器**，使两个编码器对同一组信息比特产生统计上独立的校验序列。
- 典型码率：1/3（系统比特 + 两个校验比特流）。

### 8.5.2 迭代译码
Turbo码的灵魂在于其译码器：两个**软输入软输出**译码器（如基于MAP算法的BCJR译码器）**迭代交换“外信息”**。

- **外信息**：一个译码器基于其校验比特和来自另一个译码器的先验信息，计算出的关于每个信息比特的可靠性增量。
- **迭代过程**：译码器1为译码器2提供先验信息，译码器2再为译码器1提供更新后的先验信息，如此反复。随着迭代进行，比特判决的可靠性不断提高。
- **交织器的双重作用**：在编码时打散信息序列，在迭代译码时打散错误，防止两个译码器对同一组错误陷入确认偏差。

### 8.5.3 性能与影响
Turbo码在AWGN信道下，在信噪比距离香农极限仅0.7 dB处可实现 \(10^{-5}\) 的误码率。它被3G和4G（LTE）标准采纳，证明了迭代译码思想的强大威力。

## 8.6 低密度校验码

LDPC码是另一类可逼近香农极限的强大编码，其理论提出早于Turbo码，但受限于当时的计算能力，在Turbo码之后才被重新发现并广泛应用。

### 8.6.1 结构与表示
LDPC码是一种线性分组码，其**校验矩阵 \(H\) 是“稀疏的”**（即其中“1”的密度很低）。这可以用一个**二分图**（又称Tanner图）表示：
- **变量节点**：对应码字比特。
- **校验节点**：对应校验方程。
- **边**：连接参与该校验方程的变量节点。

### 8.6.2 置信传播译码
LDPC码采用基于Tanner图的**消息传递算法**进行迭代译码。在每次迭代中：
1.  **变量节点处理**：每个变量节点收集来自信道和其他相连校验节点的信息，计算并向上传递其比特为0或1的置信度（通常用对数似然比LLR表示）。
2.  **校验节点处理**：每个校验节点根据收到的来自相连变量节点的信息，判断该校验方程是否满足，并向下传递更新后的信息。

经过多次迭代，变量节点的LLR值趋于稳定，据此做出硬判决。

### 8.6.3 优势与应用
- **性能**：在长码长时，性能甚至优于Turbo码，更接近香农极限。
- **并行性**：消息传递算法具有天然的并行结构，利于高速硬件实现。
- **灵活性**：码率和码长可灵活设计。
凭借这些优势，LDPC码已被Wi-Fi、5G eMBB数据信道、卫星电视等标准广泛采用。

## 8.7 编码调制：联合优化

传统上，编码和调制是分开设计的。**编码调制**将两者联合优化，旨在不增加带宽的前提下同时获得**编码增益**和**调制增益**。

### 8.7.1 网格编码调制
- **Ungerboeck提出**，其核心是**集分割映射**：将调制星座图逐级划分为子集，使子集内的最小欧氏距离不断增大。
- 将信息比特分为两部分：一部分用于选择星座子集（产生冗余，带来编码增益），另一部分用于在选定子集中选择星座点。
- 通过一个简单的卷积码控制子集的选择，在接收端用维特比译码。TCM在不增加带宽和功率的情况下，可获得3-6 dB的增益。

### 8.7.2 比特交织编码调制
- 更适用于衰落信道的现代方案。结构为：信道编码 + **比特级交织器** + 高阶调制。
- **核心思想**：交织器将编码比特充分打散，使得映射到同一调制符号上的几个比特具有近乎独立的可靠性。接收端进行**迭代解调与译码**。
- 在BICM-ID中，译码器将外信息反馈给解调器，帮助其改进对每个比特的软信息估计，从而获得额外增益。

## 8.8 衰落信道中的编码与交织

在衰落信道中，编码的主要目标是获取**时间或频率分集**，而不仅是纠正随机错误。

### 8.8.1 交织器设计
交织器是编码应用于衰落信道的关键。它将编码比特序列的顺序重新排列，使得：
- 在时间上相邻的编码比特在时域上被分开的距离大于信道的相干时间。
- 在频率上相邻的编码比特在频域上被分开的距离大于信道的相干带宽。
这样，一个深衰落导致的一串连续错误，在解交织后，就变成了译码器可以处理的随机分散的错误。

### 8.8.2 编码的分集阶数
在慢衰落信道中，一个码的性能由其能达到的**分集阶数**决定。理论分析表明，一个码的分集阶数等于其**码字间的最小汉明距离** \(d_{\min}\)。为了获得高分集阶数，需要设计具有较大 \(d_{\min}\) 的码，或者通过**增加编码约束长度**来实现。

## 8.9 不等差错保护编码

在许多多媒体业务中，信源数据中不同部分的重要性不同。UEP编码为更重要的比特提供更强的保护。

- **实现方式**：
  1.  **分层调制**：将重要比特映射到调制符号中更可靠的比特位。
  2.  **速率兼容穿孔卷积码**：对重要比特使用低穿孔率（高保护），对次要比特使用高穿孔率（低保护）。
  3.  **多级编码/UEP-LDPC**：设计具有非均匀校验节点度分布的LDPC码，高重要性比特对应高保护度的变量节点。

## 8.10 信源信道联合编码

根据香农的分离定理，在点对点通信中，信源编码和信道编码可以分开最优设计。但在**实际约束**和**非平稳无线信道**下，联合设计可能更优。

### 8.10.1 联合信源信道编码
- **思想**：根据信源的特性和信道状态，动态分配信源编码和信道编码的比特/冗余，实现端到端失真最小化。
- **应用场景**：
  - 图像/视频传输：在信道差时，发送更粗糙但更健壮的图像版本。
  - 语音传输：对不同的语音帧类型（清音、浊音、静默）提供不同级别的保护。

### 8.10.2 跨层设计
JSCC是更广泛的**跨层设计**思想的一部分。在无线网络中，通过联合优化物理层（调制编码）、链路层（ARQ）和应用层（信源编码、失真容忍），可以在时变、不可靠的无线链路上实现最佳的QoS。

## 8.11 总结

本章系统阐述了无线信道编码的技术体系：

1.  **演进脉络**：从线性分组码、卷积码，到革命性的Turbo码和LDPC码，编码技术不断逼近香农极限。

2.  **核心对抗技术**：
    - **交织**是应对突发错误的必备工具。
    - **迭代译码**是Turbo码和LDPC码高性能的灵魂。
    - **软信息处理**贯穿于所有现代编码方案，是获取增益的关键。

3.  **设计范式的转变**：
    - 从**纠随机错误**到**获取分集**。
    - 从**编码与调制分离**到**编码调制联合优化**。
    - 从**分层固定设计**到**跨层自适应与联合设计**。

4.  **标准中的应用**：
    - 2G/3G：卷积码为主。
    - 3G/4G：引入Turbo码。
    - 4G/5G/Wi-Fi：LDPC码成为高速数据信道的主流。

无线信道编码是连接物理层传输与高层可靠服务的桥梁。它通过智能地引入和利用冗余，将恶劣的、不可靠的无线信道转化为一个“虚拟的”可靠比特管道，从而支撑起丰富多彩的无线应用。理解编码原理，是设计和优化现代无线通信系统的基石。
